# 内容

TCG BattleSpirits のデッキシミュレータ

# ルール仕様

- ゲームフォーマット: エターナル

# 操作の例

- 初期局面:

  - リザーブコア: 4
  - トラッシュコア: 0
  - フィールド: スピリット1(シンボル:赤1, コア:1)
- 操作: スピリット2(コスト:1, シンボル:緑1, 軽減:緑1)
- 局面

  - リザーブコア: 2
  - トラッシュコア: 1
  - フィールド: スピリット1(シンボル:赤1, コア:1), スピリット2(シンボル:緑1, コア:1)
- 操作: スピリット3(コスト2: シンボル:赤1, 軽減:赤1)召喚/維持コア1個
- 局面

  - リザーブコア: 0
  - トラッシュコア: 2
  - フィールド: スピリット1(シンボル:赤1, コア:1), スピリット2(シンボル:緑1, コア:1), スピリット3(シンボル:赤1, コア:1)

# アプリ仕様

- 局面に応じた選択肢とその場合の次局面を列挙する関数
- 局面から評価値を算出する関数
  - 評価値優先順:
    - 相手のライフを減らす
    - 相手のスピリット等を破壊する
    - 召喚スピリット数
  - 非公開領域(互いのデッキ、相手の手札)は評価に使用しない

## AI評価仕様 (Burn強化学習)

AI強化学習基盤として、以下の関数仕様を策定し実装する。

1. **特徴量エンコーダ**
   - 関数: `fn encode_state(state: &GameState) -> Vec<f32>`
   - 役割: `GameState` をBurnが処理できる一次元の数値テンソルに変換する。
   - 内容: フェイズのOne-Hot化、コア数の正規化(0.0〜1.0)、フィールドオブジェクトの特徴量（コスト、レベル、シンボル、疲労状態）等の数値化。

2. **盤面評価ネットワーク (MLP)**
   - 構造体: `BoardEvaluator<B: Backend>`
   - 関数: `fn forward(&self, input: Tensor<B, 2>) -> Tensor<B, 2>`
   - 役割: エンコードされた盤面を入力とし、現在のプレイヤーから見た勝率予測値 `Value` (例: -1.0〜1.0 または 0.0〜1.0) を出力する。

3. **学習データ収集 (Self-Play)**
   - 関数: `fn generate_self_play_data(model: &BoardEvaluator<B>) -> Vec<Experience>`
   - 役割: AI同士を対戦させ、各ターンの「状態テンソル」と最終的な「勝敗(報酬)」をセットにした学習データを生成する。

4. **学習ループ (Training Step)**
   - 関数: `fn train_model(data: Vec<Experience>, epochs: usize)`
   - 役割: `Burn` の `Learner` を用い、収集したExperienceバッファを元にネットワークの重み(Weights)を最適化する（Loss関数の最小化）。

- ログファイル:
 - 局面情報を(bs-log.yaml)に時系列で保存
 - カード(スピリット情報ではない)情報はカード番号のみ
```
- {tuen:1, phase: startStep, player1: { ... }, player2: {...} }
- {tuen:1, phase: coreStep,  player1: { ... }, player2: {...} }
- {tuen:1, phase: drawStep,  player1: { ... }, player2: {...} }
- {tuen:1, phase: refreshStep,  player1: { ... }, player2: {...} }
- {tuen:1, phase: mainStep,  player1: { ... }, player2: {...} }
...
```

- 初期配置

  - 定義ファイルに基づきデッキ配置（引数により任意のデッキパスを指定可能）
- 局面においての選択肢を列挙する列挙関数（重複アクションの完全排除）
- 選択に基づき次の局面を生成する遷移関数（コスト支払い・移動・消滅およびライフ減少の自動処理）
- 初期状態から、下記を繰り返す:

  - 局面に応じた選択肢を列挙し、ユーザによる選択を入力（nキーによるステップ終了/パス、qキーによるサレンダー対応）
    - 選択肢がない(n,qのみ)の場合は選択は行わずその選択肢を実施、その旨表示する
  - 入力に応じ、次の局面を生成する
- 表示

  - コア
    - コア3個(ソウルコアなし): "3"
    - コア3個(うち1個ソウルコア): "3s"
  - トラッシュコア
    - コスト支払いやコア移動に応じて、1コアの不整合なく正確に加減算および同期表示される仕様
  - オブジェクト:
    - 回復状態: 表示なし
    - 疲労状態: "[疲労]"
  - スピリット: `名前(シンボル:赤1, コア:3)[疲労]`
  - 手札: `名前 (コスト:X, 軽減:シンボル)` （例: `天雷の契約神インドラ (コスト:3, 軽減:黄2)`）カード本来の軽減シンボルを表示
  - 選択肢:
    - 第1階層（カテゴリ選択）: `【手札から使用/召喚】 名前 (コスト:X, 軽減後:Y)`
    - 第2階層（詳細コア支払）: `コスト2s, 配置コア:1, (リザーブ:1s, スピリット1:2)`
      - リザーブからの支払いが多いものを上位に表示
      - ソウルコアをコストとしてトラッシュに送るものを下位に表示
    - 元コストと、現在のフィールドシンボルから算出した自動軽減適用後の支払コストを両表示する仕様
    - 選択肢それぞれに評価値を付与
- 起動方法
  - 常に対話型CLI（対話型対戦シミュレーター）として起動（AI自立対戦シミュレーション機能は廃止）
